Quick Launch & Onboarding Guide

Ready to deploy? Follow this step-by-step setup to execute the first-time onboarding system, check system health, and boot the web dashboard.

Step 1: One-Line Package Installation

Install the repository dependencies using pip to register the global executable on the local system:

# Install directly in editable mode
pip install -e .

# Or install optional speech-to-text / voice dependencies
pip install -e .[voice]

Step 2: Execute First-Time Onboarding

Just like the reference setup scripts, run the onboarding tool to automatically generate the local workspace folder structure, system settings, and SQLite tables:

cyberclaw onboard

📁 WHAT ONBOARDING CREATES:

  • `config.user.yaml` ➔ Default YAML config containing templates for channels, voice, and system credentials.
  • `.secrets/` ➔ The folder structure for secure local vaults.
  • `history.db` & `tasks.db` ➔ The core SQL databases for storing your chat history and running background jobs.

Step 3: Run System Diagnostics (The Doctor Check)

Run the diagnosis command to verify permissions, config schemas, active databases, and key encryption flags:

cyberclaw doctor

This command runs 10+ validation checks and prints a beautiful terminal checklist. If there is a configuration error, it tells you exactly how to fix it!

Step 4: Secure Your Vault

CyberClaw uses cryptography to store your LLM API keys. Encrypt your API key with the secrets store:

cyberclaw secrets set OPENAI_API_KEY "your-openai-api-key-here"
cyberclaw secrets set DEEPGRAM_API_KEY "your-deepgram-api-key-here"

Step 5: Launch Interactive Web Control Center

Launch the background gateway server to host the Web UI Control Panel:

cyberclaw gateway start

💡 TIP

Open your browser and navigate to http://localhost:8000/ui. You will be greeted by the premium Dark Space Web UI Control Dashboard where you can monitor active channels, search live models, check running tasks, and chat directly!

The 17-Command Reference Masterclass

CyberClaw features a powerful console command center. Here is the exhaustive reference tutorial for all 17 commands available in the CLI.

1. `onboard`

Initializes a fresh workspace directory structure, configuring local templates, assets, and files necessary for first-run execution.

cyberclaw onboard

2. `doctor`

Runs self-diagnostic protocols, checking configurations, local sqlite access, environment API variables, and AES credentials.

cyberclaw doctor

3. `chat`

Starts an interactive CLI session where you can chat directly with your AI in real-time, test system tools, and view response times.

cyberclaw chat

4. `agent`

Runs a single, non-interactive, turn-based query for command-line batch piping or developer automated tasks.

cyberclaw agent --message "Scrape the README file and print the results"

5. `server`

Starts the core 24/7 background process execution layer, loading hot-load schedules, cron managers, and message busses.

cyberclaw server

6. `gateway`

Manages API gateways, starts background web servers, and hosts the beautiful Web UI Control panel.

# Start the local gateway and host the Web UI
cyberclaw gateway start --port 8000

# Stop the gateway server
cyberclaw gateway stop

7. `config`

Inspects or updates the current settings dynamically. Prints out the configuration values cleanly.

# Show active configuration settings
cyberclaw config show

# Set workspace values programmatically
cyberclaw config set workspace.path "./my_workspace"

8. `secrets`

Interactive manager for adding, removing, and listing encrypted credentials. All values are securely locked.

# Securely encrypt an API key
cyberclaw secrets set DEEPGRAM_API_KEY "token"

# List vaulted keys
cyberclaw secrets list

9. `sessions`

Inspects active conversational logs, titles, session tokens, and database indices directly from the console.

cyberclaw sessions list

10. `pairing`

Manages messaging pairing locks for Telegram, Discord, and WhatsApp, keeping external access secure.

# Approve an inbound pairing code
cyberclaw pairing approve --code "123456"

11. `channels`

Lists all configured communications pipelines, showing connection indicators, REST endpoints, and statuses.

cyberclaw channels list

12. `providers`

Lists available API backends (OpenAI, Gemini, Groq, Anthropic) without revealing any secret keys.

cyberclaw providers list

13. `talk`

Launches active streaming voice mode, listening for wake-word keywords or spacebar fallback commands.

cyberclaw talk

14. `migrate-history`

Converts legacy JSONL local files into robust SQLite database tables, preserving conversation records with ACID protection.

cyberclaw migrate-history

15. `service`

Registers and controls CyberClaw as a persistent background service on Windows operating systems.

# Install background service
cyberclaw service install

# Start background service execution
cyberclaw service start

16. `update`

Queries online updates and upgrades your local environment automatically using the uv package runner.

cyberclaw update

17. `version`

Displays the current version, build signature, and workspace configurations.

cyberclaw version

Model Catalog System

Learn how to manage the global LLM model library, calculate accurate pricing costs, and query the cheapest or most capable engines.

Overview

CyberClaw holds a local model catalog tracking pricing values per million tokens (input/output), context windows, capabilities (reasoning, vision, tool use), status (available/preview/deprecated), and auto-replacements.

Common Catalog Code Operations

Query or find appropriate models in Python using these simple operations:

from cyberclaw.core.model_catalog import get_catalog

# Initialize central registry
catalog = get_catalog()

# Estimate token usage cost in USD
cost = catalog.estimate_cost("openai", "gpt-4o-mini", input_tokens=1500, output_tokens=800)
print(f"Call Cost: ${cost:.6f} USD")

# Query the cheapest model that supports vision
cheapest_vision_model = catalog.find_cheapest(needs_vision=True)
print(f"Selected Model: {cheapest_vision_model.ref}")

Live Model API Discovery

You can discover models on-the-fly directly from any standard OpenAI-compatible API endpoint:

# Query models from Groq or OpenRouter live API using your credential
await catalog.discover_live(provider="groq", api_key="your-api-key")

Task System & Database Schema

Master background task workers and explore the ACID-protected SQLite schema holding tasks and conversation histories.

Database Locations

CyberClaw holds two dedicated database directories under your workspace folder:

  • History Store: workspace/history.db
  • Task Registry: workspace/.tasks/tasks.db

The SQLite Task Schema

Below is the exact database schema configuration defining background tasks:

CREATE TABLE tasks (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    agent_id TEXT NOT NULL,
    instruction TEXT NOT NULL,
    status TEXT NOT NULL DEFAULT 'pending',
    priority TEXT NOT NULL DEFAULT 'normal',
    result TEXT DEFAULT '',
    error TEXT DEFAULT '',
    created_at REAL NOT NULL,
    started_at REAL DEFAULT 0,
    completed_at REAL DEFAULT 0,
    timeout_seconds INTEGER DEFAULT 300,
    retry_count INTEGER DEFAULT 0,
    max_retries INTEGER DEFAULT 1,
    metadata TEXT DEFAULT '{}'
);

Running a Task Programmatically

from cyberclaw.core.tasks import TaskStore, TaskExecutor
from pathlib import Path

# Connect to database and submit tasks
store = TaskStore(Path("./workspace/.tasks/tasks.db"))
executor = TaskExecutor(store, max_concurrent=3)

task = await executor.submit(
    name="Log analyzer",
    agent_id="cyberclaw",
    instruction="Analyze production error records from database"
)
print(f"Task submitted: {task.id}")

Context Engine

Discover how CyberClaw dynamically builds token-budgeted prompt payloads using plugin-contributed data feeds.

Dynamic Context Injection

The ContextEngine automatically collects context updates from multiple sources, sorts them by priority, and ensures the payload fits perfectly inside the context window constraint before querying the LLM provider.

Registering a Custom Context Feed

Add a new system context provider in your backend plugins:

from cyberclaw.core.context_engine import ContextEngine, ContextRequest, ContextContribution

engine = ContextEngine()

# Register a prioritized feed that injects the current timestamp
def system_time_provider(request: ContextRequest):
    import time
    return ContextContribution(
        source="time_agent",
        priority=100, # High priority, injected first
        content=f"Current System Time: {time.strftime('%Y-%m-%d %H:%M:%S')}"
    )

engine.register_provider("time", system_time_provider)

Auto-Reply & External Channels

A step-by-step tutorial on connecting CyberClaw to Telegram, Signal, WhatsApp, or Discord and managing rules.

1. Signal Rest Bridge Integration

The Signal channel integrates securely via `signal-cli-rest-api`. Configure the endpoints in your active configurations:

# config.user.yaml
channels:
  signal:
    enabled: true
    rest_url: "http://localhost:8080"
    phone_number: "+1234567890"

2. Telegram Bot Integration

Create a bot via BotFather, get the bot token, and update the secrets store:

cyberclaw secrets set TELEGRAM_BOT_TOKEN "your-bot-token"

3. Discord Bot Integration

Create an application on Discord Developer Portal, configure bot intents (Message Content intent is required), and add the bot token:

cyberclaw secrets set DISCORD_BOT_TOKEN "your-bot-token"

WebSocket Transcription (Deepgram)

CyberClaw supports real-time streaming speech-to-text. Audio bytes are sent live over WebSockets for rapid transcriptions:

from cyberclaw.voice.realtime_stt import RealtimeTranscriptionManager

mgr = RealtimeTranscriptionManager()

# Start receiving transcribed segments
await session.start(on_transcript=lambda segment: print("Transcribed:", segment.text))

Security & Cryptography

Learn how CyberClaw protects sensitive corporate records, API keys, and messaging pipelines.

AES Symmetric Secrets

API keys are securely vaulted in vault.json using Fernet symmetric encryption. The key is dynamically derived from your local machine's unique hardware signature.

Configuring the Vault

# Add a secret safely
cyberclaw secrets set OPENAI_API_KEY "sk-..."

# Remove a secret safely
cyberclaw secrets delete OPENAI_API_KEY

# List all securely vaulted variables (redacts sensitive characters)
cyberclaw secrets list

Services & Scaling

Learn how to scale CyberClaw for production, monitor health metrics with Prometheus, and update files.

Running as a Windows background Service

You can run CyberClaw as a 24/7 background Windows Service using standard NSSM controls:

# Install the CyberClaw background service
cyberclaw service install

# Start the background service
cyberclaw service start

# Check the status of the service
cyberclaw service status

Observability with Prometheus Metrics

CyberClaw tracks real-time usage metrics (inference latency, token throughput, uptime, and health stats). Retrieve them as Prometheus metrics or JSON telemetry data:

# Query Prometheus metrics
curl http://localhost:8000/metrics

# Query structured JSON metrics
curl http://localhost:8000/metrics/json

These values can be integrated directly with standard Grafana dashboards for production monitoring.
